# Governance

## Decision Making

This project is maintained by the Appstrate core team. Decisions about features,
architecture, and releases are made by the maintainers listed in MAINTAINERS.md.

## Contributions

All contributions are welcome via pull requests. Maintainers review and merge PRs
based on code quality, test coverage, and alignment with the project roadmap.

## Releases

Releases follow semantic versioning. The core team decides when to cut releases
based on the changelog and the stability of accumulated changes.

## Changes to Governance

Changes to this governance model are proposed via pull request and require
approval from at least two maintainers.
